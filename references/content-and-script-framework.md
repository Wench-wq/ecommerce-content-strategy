# 抖音内容策略与脚本

## 内容矩阵

从痛点解决、产品证明、食用场景、口味口感、配料工艺、用户口碑、对比测评、热点借势、情绪共鸣和促销转化中选择与项目证据匹配的支柱。

每个支柱绑定目标人群、用户问题、SKU、卖点、证据、内容形式和转化动作。先覆盖购买决策链，再扩展数量。

## 选题优先级

按用户需求强度、差异化、证据强度、可视化程度、转化距离、拍摄成本和合规风险评分。优先制作需求强、证据足、能清晰展示且靠近成交的选题。

## 完整脚本格式

每条脚本包含：

1. 标题与内容目标
2. 目标人群、对应SKU和核心卖点
3. 建议时长及发布场景
4. 前3秒钩子
5. 按时间段列出的镜头、景别、画面动作、口播、字幕、音效/音乐
6. 产品或证据特写
7. CTA
8. 合规注意事项

优先写能被拍摄执行的具体动作和画面。避免“高级感镜头”“展示产品优势”等空泛指令。

## 脚本节奏

- 0–3秒：明确人群、问题、反差或结果，不做冗长铺垫。
- 3–10秒：展示问题或感官刺激，并让产品自然出现。
- 中段：用配料、工艺、演示、对比或评论提供证明。
- 结尾：明确适合谁、怎么买、选哪个SKU以及下一步动作。

对比内容使用可验证的同口径信息，不贬损竞品。评论内容注明是个体体验，不能把体验升级为医疗或普遍功效。
