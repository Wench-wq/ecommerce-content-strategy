# 食品饮料产品分析

## 产品事实清单

提取品类/子品类、配料表、营养成分、净含量、口味、形态、工艺、产地、包装、储存、保质期、食用方法、适用及不适用人群、检测或认证材料。

不要仅凭包装正面文案推断配料、营养或功效。出现冲突时优先采用法规标签、检测材料或用户确认的信息，并记录冲突。

## 卖点结构

使用“事实 → 消费者利益 → 场景 → 证明”的链条：

| 层级 | 需要回答的问题 |
|---|---|
| 产品事实 | 产品客观上有什么？ |
| 感官/功能利益 | 消费者吃喝时得到什么直接体验？ |
| 情绪利益 | 为什么值得期待、分享或复购？ |
| 场景利益 | 在什么时间、地点和任务中更有价值？ |
| 证明 | 标签、工艺、演示、检测或真实反馈如何支持？ |

## 食品饮料常用分析维度

- 口味：甜度、酸度、咸度、香气、层次、余味及刺激感。
- 口感：脆、糯、酥、绵、稠、顺滑、颗粒感、气泡感及温度体验。
- 配料：核心原料、排序、添加物、过敏原及消费者易误解点。
- 便利：开封、携带、冲泡、分装、保存和清洁成本。
- 场景：早餐、通勤、办公室、健身、夜宵、聚会、送礼、亲子和户外。
- 价值：每份价格、每次食用成本、规格匹配和浪费风险。
- 信任：产地、供应链、生产标准、检测、标签完整度和品牌背书。

## 定位输出

输出一句话定位：`为[目标人群]在[关键场景]提供[核心利益]的[子品类]，凭借[可信差异点]区别于[替代选择]。`

再输出3–5个核心卖点。每项包含短标题、消费者语言、事实依据、适用场景和表达风险。
